# LUT 行业工作流调研

> 日期: 2026-06-16 | 来源: colour-science 文档、NVIDIA GPU Gems 2、Adobe .cube Spec、专业调色社区

## 典型工作流

```
拍摄(Log) → 技术转换(Log→709) → 一级校正 → 创意LUT → 精修 → 输出
              │                    │          │
         技术LUT/CST          曝光/白平衡   风格LUT (60-80%强度)
```

## LUT 类型

| 类型 | 用途 | 顺序 |
|------|------|------|
| 技术 LUT | log→709 色彩空间转换 | 必须先于创意 LUT |
| 创意 LUT | 胶片模拟/风格化色彩 | 输入必须是已正常化画面 |
| 混合 LUT | 技术+创意打包 | 不推荐（无法独立调整） |

## .cube 格式

- 33³ = 35,937 行 RGB 浮点三元组
- 1D LUT: 每通道独立，只能调 gamma/对比度
- 3D LUT: 通道间可交叉（色相旋转、色域映射、胶片质感）
- OpenCV 的 `cv2.LUT()` 只支持 1D LUT

## Python 实现方案

| 方案 | 适用场景 |
|------|---------|
| `colour.read_LUT()` + `.apply()` | 推荐 — 一行解析，一行应用 |
| 手写解析 + numpy 三线性插值 | 教学/定制需求，约 30 行 |
| `pycubelut` CLI | 轻量级命令行 |
| GPU 3D 纹理 (OpenGL/Vulkan) | 实时视频预览 |

## 与 ProjectLUT 的关系

- 152 个 LUT 全为创意/风格 LUT
- 目录中 `log` 子目录 → 需先做 log→709 技术转换
- 目录中 `709` 子目录 → 已正常化，直接套用
- processor.py 需判断色彩空间再决定是否做技术转换
